## 5. 참고 — 031a(시드 데이터, 36~38일차) / 030(프리시즌, 54~62일차) / 043(관측성, 39~42일차)

- **031a 시드 데이터**: 36그룹 실제 기본값은 **데이터**이며 타입 변경 대상이 아니다(위 §1 근거와 동일). `config.ts`의 `CommonCode`/`CommonCodeGroup`이 그대로 그릇 역할을 한다.

- **043 관측성**: NFR-OB-001(상관 ID 로깅)·NFR-OB-004(헬스체크)에 필요한 `CronRun`/`CronGap`/`AuditLog`(E-45~47)는 **오늘(8일차) 1팀이 이미 `ops.ts`에 추가**했음을 확인했다(파일 헤더 주석: "1차 범위 잔여분, 8일차 반영"). NFR-OB-002의 "6종 메트릭"(시뮬 소요·라운드 지연·크론 성공률·배당 산출 시간·정산 시간/실패율·API 지연)은 `docs/require/05-data-requirements.md`의 E-01~E-47 엔티티 목록에 없는 **집계/파생 지표**이며, `CronRun.durationMs` 등 원자료로부터 산출하는 로직(`src/lib/obs/metrics.ts`, 3팀 Task 040) 소관이지 도메인 타입 신설 대상이 아니라고 판단했다.

## 6. 종합 판정 (팀장 전달용)

### ② 후속 배치 반영으로 미뤄도 되는 것
- **Manager 대행(acting) 여부 구분 플래그 부재** (근거: D-23) — 030 프리시즌(Task 059, 54일차 이후) 착수 전 이슈 등록 → 배치 반영으로 충분. 003/007/029/035 어디에도 필요하지 않음.
