| 필드 | 내용 |
|---|---|
| **요구사항** | 동률 시 다음 순서로 순위를 결정한다.<br>**1. 승점 → 2. 골득실(GD) → 3. 다득점(GF) → 4. 승자승(동률 팀들만의 미니리그: 상호 승점 → 상호 골득실 → 상호 원정 다득점) → 5. 다승(총 승수) → 6. 페어플레이 점수(경고 1점 / 경고누적퇴장 3점 / 직접퇴장 4점, 낮을수록 상위) → 7. 월드 시드 기반 결정론적 추첨**<br>**예외**: 승격 경계(3위/4위) 또는 강등 경계(N−2위/N−3위)에서 6단계까지 동률이면 → **중립지 단판 플레이오프**(연장 30분 → 승부차기)로 결정한다. |
| **우선순위** | M |
| **수용 기준** | ① 7단계 각각에 대해 해당 단계에서만 순위가 갈리는 시나리오 단위 테스트 통과<br>② 7단계 추첨은 시드 고정 시 항상 동일 결과 반환<br>③ 승강 경계 동률 발생 시 `competition_type = TIEBREAK` Fixture가 자동 생성되고 UI에 노출<br>④ 어떤 입력에도 순위가 확정되어 무한 루프가 발생하지 않음 |
| **상태** | [확정] |
| **근거** | 1~3단계는 프리미어리그식(단순·직관), 4~5단계는 라리가식(직접 대결 존중)을 혼합. 자동 진행 세계이므로 무한 동률 루프를 반드시 종결시켜야 하여 6·7단계를 최종 결정자로 둠. 다만 승강 경계는 서사·배팅 상품 가치가 크므로 예외적으로 실경기를 부여. |

### FR-LG-009 · 시즌 종료 시점 정렬 및 라운드 간격 (원본 미해결 해소 / D-01 반영)

| 필드 | 내용 |
|---|---|
| **요구사항** | 리그별 라운드 수가 달라 동일 간격이면 종료 시각이 벌어진다. 이를 **리그별 라운드 간격 차등**으로 해결한다.<br>**기본값 (공통코드 `ROUND_INTERVAL_MIN`)**<br>· 리그1: 46라운드 × **75분** = 3,450분<br>· 리그2: 38라운드 × **90분** = 3,420분<br>· 리그3: 30라운드 × **115분** = 3,450분<br>정규시즌 총 길이 ≈ **3,450분 = 57.5시간 ≈ 2.4일**<br>각 리그의 **최종 라운드는 T+3,450분에 강제 정렬**하여 잔여 오차를 흡수한다. |
| **우선순위** | M |
| **수용 기준** | ① 세 리그 최종 라운드 킥오프 시각 차이 ≤ **30분**<br>② 각 리그 라운드 간격이 공통코드 값과 일치하며 하드코딩되지 않음<br>③ 배속(`speed_multiplier`) 변경 시 세 리그 간격이 동일 비율로 스케일되고 동시 종료 정렬이 유지됨<br>④ 공통코드로 간격을 변경하면 **다음 시즌부터** 반영되고 진행 중 시즌 스케줄은 불변 |
| **상태** | [확정] |
| **근거** | 원본의 "15분마다 1경기 킥오프"를 **경기 단위**로 해석하면 리그1 시즌이 552×15분 = 138시간이 되고 리그 간 종료 격차가 78시간에 달해 서비스로 성립하지 않는다. **라운드 단위 동시 킥오프 + 리그별 간격 차등**이 (a) 종료 시점 정렬 (b) 라이브 화면의 동시 다경기 연출 (c) 라운드 단위 순위 갱신을 모두 만족하는 유일한 해법이다.<br>간격 비율 15:18:23은 유지하되, 사용자 결정(D-01)에 따라 약 5배 완만화하여 75:90:115로 확정했다. |

### FR-BT-005 · 배당 산출

| 필드 | 내용 |

| **요구사항** | 시뮬레이션 엔진 자체를 몬테카를로로 반복 실행해 결과 분포를 얻고 확률 → 배당으로 변환한다.<br>· 경기 마켓: 킥오프 T−30분에 **N = 3,000회** 프리시뮬<br>· 시즌 마켓: 라운드마다 잔여 시즌 **N = 300회**<br>· 배당 = `(1 / p) × (1 / overround_factor)`, **오버라운드 마진 6%** (`Σ(1/odds) = 1.06`)<br>· 최소 1.01, 최대 500.00<br>· 확률 0 셀렉션은 마켓에서 제외<br>N·마진·상하한은 전부 공통코드 `ODDS_*`로 관리한다. |
