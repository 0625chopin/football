## 4.1 NFR-PF — 성능

|---|---|---|---|---|

| **NFR-PF-008** | 공개 API 응답 시간 | M | 목록·상세 조회 **p95 ≤ 300ms**, **p99 ≤ 800ms** | [확정] |
| **NFR-PF-009** | 첫 화면 로드 (4G, 중급 모바일) | M | **LCP ≤ 2.5s**, **CLS ≤ 0.1**, **INP ≤ 200ms** | [확정] |

| **NFR-PF-014** | 상수 스냅샷 저장 오버헤드 | M | 경기 1건당 추가 지연 **≤ 5ms** (해시 중복 제거로 대부분 참조만 발생) | [확정] |

## 4.2 NFR-DT — 결정론 / 재현성

| ID | 요구사항 | 우선순위 | 측정 가능한 수용 기준 | 상태 |
|---|---|---|---|---|
| **NFR-DT-001** | 전 시뮬레이션은 시드 기반 PRNG만 사용하며, 확률 판정에 `Math.random()` / `Date.now()` 를 사용하지 않는다 | M | ESLint `no-restricted-globals` 커스텀 룰 위반 **0건**. CI에서 위반 시 빌드 실패 | [확정] |
| **NFR-DT-002** | 동일 `match_seed` + 동일 입력 상태 + 동일 상수 스냅샷 → 이벤트 로그 **100% 동일** | M | Vitest 골든 스냅샷 **100경기** 비교 전건 일치 | [확정] |
| **NFR-DT-003** | 동일 `world_seed` → 5시즌 진행 후 전 엔티티 상태 해시 동일 | M | 상태 해시(정렬된 엔티티 직렬화의 SHA-256) 비교 일치 | [확정] |
| **NFR-DT-004** | 시드가 계층 파생되어 특정 경기만 재현할 때 전체 재실행이 불필요하다 | M | 단일 경기 재현 **≤ 100ms** | [확정] |
| **NFR-DT-005** | 부동소수 연산 순서 의존성을 제거하기 위해 확률 판정 임계값 비교를 **고정 정밀도(소수 6자리 반올림)** 로 수행한다 | S | Node 버전 2종 × OS 2종에서 동일 시드 결과 일치 | [확정] |
| **NFR-DT-006** | 배당 산출용 프리시뮬은 본경기와 **독립된 시드 네임스페이스**를 사용해 프리시뮬 결과로부터 본경기 결과를 역산할 수 없다 | M | ① 프리시뮬 시드 ≠ 본경기 시드 검증<br>② 프리시뮬 최빈 결과와 본경기 결과의 상관계수가 확률적 기대치를 넘지 않음(1,000경기 검증) | [확정] |
| **NFR-DT-007** | **상수 외부화가 결정론을 깨지 않는다** — 모든 시뮬 결과는 시드 + 상수 스냅샷 쌍으로 재현 가능하다 | M | ① 공통코드를 변경한 뒤 과거 경기를 재현해도 결과 100% 일치 (FR-AD-014)<br>② `snapshot_id`가 null인 `fixture` 0건 | [확정] |
| **NFR-DT-008** | 집합·맵 순회 순서에 의존하지 않는다. 정렬이 필요한 곳은 항상 명시적 tiebreak 키를 포함한 안정 정렬을 사용한다 | M | ① 정렬 호출부 코드 리뷰 체크리스트 통과<br>② 입력 순서를 셔플해도 동일 결과를 내는 Vitest 테스트 통과 | [확정] |

## 4.3 NFR-QA — 검증 / 테스트 (Vitest) 【D-03 반영】

> **결정 D-03**: Vitest 도입 확정. 기존 초안의 "테스트 러너 부재 → tsc + 골든 파일로 대체" 우회안은 폐기한다.
> 단, **본 요구사항 단계에서 패키지를 설치하지 않는다.** 설치·구성은 구현 단계에서 수행한다.

| ID | 요구사항 | 우선순위 | 측정 가능한 수용 기준 | 상태 |
|---|---|---|---|---|
| **NFR-QA-001** | 테스트 러너로 **Vitest**를 도입한다. `npm run test`, `npm run test:watch`, `npm run test:coverage` 스크립트를 제공한다 | M | ① 3개 스크립트가 동작<br>② 설정 파일(`vitest.config.ts`)이 `@/*` 경로 별칭을 해석<br>③ CI에서 `npm run test` 실패 시 머지 차단 | [확정] |
| **NFR-QA-002** | **엔진 순수 함수 단위 테스트** — `src/lib/sim/` 의 모든 계수 함수·판정 함수·집계 함수에 단위 테스트를 작성한다 | M | ① `src/lib/sim/` 라인 커버리지 **≥ 80%**, 브랜치 커버리지 **≥ 70%**<br>② 다음 함수는 커버리지 **100%** 필수: `conditionMultiplier`, `fitnessMultiplier`, `positionMultiplier`, `homeMultiplier`, `familiarityMultiplier`, `leagueFinishPoints`, `marketValue`, `matchRating`, `tiebreakComparator`<br>③ 각 함수의 **경계값**(하한·상한·클램프 경계)이 테스트에 포함 | [확정] |
| **NFR-QA-003** | **시드 고정 스냅샷 테스트** — 동일 시드로 실행한 결과가 저장된 스냅샷과 일치함을 검증한다 | M | ① 경기 스냅샷 **≥ 100건**(리그·컵·플레이오프·연장·승부차기 각 케이스 포함) 전건 일치<br>② 시즌 스냅샷 **≥ 3시즌**(대진표·최종 순위·승강 결과·수상) 일치<br>③ 스냅샷 갱신은 명시적 커맨드(`-u`)로만 가능하며, 무단 갱신 시 리뷰에서 지적 가능하도록 diff가 커밋에 드러남 | [확정] |
| **NFR-QA-004** | **확률 분포 불변식 테스트** — 대량 시뮬 결과의 통계 지표가 지정 밴드 내에 있음을 검증한다 | M | 10,000경기 시뮬 기준<br>① 경기당 평균 총 득점 **2.5 ~ 3.0**<br>② 홈 승률 **42 ~ 48%**, 무승부 **24 ~ 28%**, 원정 승률 **27 ~ 33%**<br>③ 경기당 평균 슈팅 **20 ~ 30**, 유효슈팅 비율 **30 ~ 40%**<br>④ 경기당 평균 경고 **3.0 ~ 5.0**, 퇴장 **0.03 ~ 0.12**<br>⑤ 0-0 무승부 비율 **6 ~ 12%**<br>⑥ 시즌 부상 발생률 스쿼드 대비 **25 ~ 45%**<br>⑦ 모든 밴드는 공통코드로 조정 가능하되, 테스트는 현재 설정값 기준으로 검증 | [확정] |
| **NFR-QA-005** | **회계 항등식 테스트** — 포인트 총량이 보존됨을 검증한다 | M | 3시즌 시뮬 기준<br>① 각 팀에 대해 `Σ(원장 수입) − Σ(원장 지출) = 잔고 변화` — 오차 **0**<br>② 각 스폰서에 대해 동일 항등식 성립<br>③ 시스템 발행 포인트(순위·상금·보조금) − 시스템 회수(없음) = 전 주체 잔고 증가 총합 + 스폰서 순유출<br>④ 이적료는 zero-sum — `Σ TRANSFER_IN + Σ TRANSFER_OUT = 0`<br>⑤ 스폰서 분배는 zero-sum — `Σ SPONSOR_SHARE(팀 −) + Σ SPONSOR_SHARE(스폰서 +) = 0` | [확정] |
| **NFR-QA-006** | **불변식(invariant) 테스트** — 세계 상태의 구조적 제약이 항상 성립함을 검증한다 | M | 10시즌 시뮬 후 매 시즌 종료 시점에<br>① 리그별 팀 수 = 24 / 20 / 16<br>② 전 팀 스쿼드 크기 22~30, GK ≥ 2<br>③ 40세 이상 현역 0명<br>④ 팀당 활성 스폰서 계약 ≤ 3<br>⑤ 계약 기간 1~5, 스폰서 계약 1~10<br>⑥ 모든 선수 OVR ≤ PA<br>⑦ 전 능력치 1~30, 컨디션 1~10, fitness 0~100<br>⑧ 스폰서 풀 크기 ≥ 40<br>⑨ 리그3 팀 수가 16 미만이 되지 않음 | [확정] |
| **NFR-QA-007** | **정산 정확성 테스트** (2차) — 마켓별 정산 규칙이 결과에 대해 올바른 판정을 내림을 검증한다 | S | ① 각 마켓 타입에 대해 최소 5개 결과 케이스 테스트<br>② 멱등성 테스트 — 정산 2회 실행 시 지갑 반영 1회<br>③ 보이드 환불 정확도 100% | [확정] |
| **NFR-QA-008** | **공통코드 로딩 테스트** — 상수가 코드가 아닌 설정에서 읽힘을 검증한다 | M | ① 공통코드 값을 테스트에서 주입했을 때 시뮬 결과가 그에 맞게 변함<br>② 값 미등록 시 기본값 폴백 + 경고 로그 발생<br>③ 발효 정책(`NEXT_SEASON` / `IMMEDIATE` / `NEXT_MARKET`)별 적용 시점 테스트 | [확정] |
| **NFR-QA-009** | **성능 회귀 테스트** — 벤치마크 결과가 기준치를 초과하면 실패한다 | S | ① `NFR-PF-001`, `NFR-PF-003`, `NFR-PF-006` 기준치를 벤치마크로 검증<br>② 기준 대비 **20% 이상 느려지면** 테스트 실패 | [제안] |
| **NFR-QA-010** | 타입 체크와 린트가 테스트와 함께 게이트를 구성한다 | M | `npx tsc --noEmit` 오류 0, `npm run lint` 경고 0, `npm run test` 통과 — 3개 모두 만족해야 머지 가능 | [확정] |

## 4.4 NFR-CFG — 설정 외부화 (공통코드) 【D-01 반영】

| ID | 요구사항 | 우선순위 | 측정 가능한 수용 기준 | 상태 |
|---|---|---|---|---|
| **NFR-CFG-001** | **코드 변경 없이 밸런싱이 가능해야 한다.** 튜닝 가능한 상수는 DB 공통코드에서 로드한다 | M | ① FR-AD-011의 30개 대상 항목이 전부 공통코드로 등록<br>② 공통코드 커버리지 **≥ 90%** (전체 튜닝 상수 대비, 분기별 감사)<br>③ 엔진 코어(`src/lib/sim/`)에 대상 상수의 숫자 리터럴 잔존 0건 (정적 검사 스크립트) | [확정] |
| **NFR-CFG-002** | **변경 이력이 추적 가능해야 한다** | M | ① 모든 공통코드 변경이 `common_code_history`에 1:1 기록<br>② 이력에 이전값·신규값·변경자·시각·사유 포함<br>③ 이력 테이블은 append-only (UPDATE/DELETE 권한 없음)<br>④ 임의 시점의 상수 값 집합을 이력으로 복원 가능 | [확정] |
| **NFR-CFG-003** | **설정 변경이 진행 중 시뮬레이션의 결정론을 깨지 않아야 한다** | M | ① 모든 `fixture`가 `snapshot_id`를 보유<br>② 상수 변경 후 과거 경기 재현 결과가 100% 일치<br>③ 진행 중 시즌의 밸런스 상수 변경이 그 시즌 경기 결과에 영향 0 (FR-AD-013) | [확정] |
| **NFR-CFG-004** | **설정 값 검증** — 잘못된 값이 저장되어 세계가 붕괴하지 않아야 한다 | M | ① 각 코드에 타입(INT/DECIMAL/STRING/BOOL/JSON)과 허용 범위(min/max) 메타데이터 존재<br>② 범위 밖 값 저장 시도 시 DB 제약 또는 애플리케이션 검증으로 거부<br>③ JSON 타입은 스키마 검증 통과 필수<br>④ 검증 실패율 0% (저장된 값 중 범위 위반 0건) | [확정] |
