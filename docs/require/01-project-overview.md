## 1.1 비즈니스 목적

컴퓨터가 전 과정을 자동으로 진행하는 **가상 축구 리그 생태계**를 실시간으로 운영하고, 사용자는 팀을 조작하지 않은 채 **관전 + 배팅**만 수행하는 웹 서비스를 구축한다.

핵심 명제:

> "플레이하지 않고, 지켜보고 예측한다."

사용자는 매니저가 아니라 **관찰자이자 예측자**다. 따라서 제품의 가치는 (a) 세계가 얼마나 그럴듯하게 굴러가는가, (b) 그 세계를 얼마나 잘 들여다볼 수 있는가, (c) 예측할 만한 불확실성이 있는가 — 이 셋에 달려 있다.

## 1.3 핵심 가치 제안

1. **살아있는 세계 (Persistent World)**
   선수는 성장하고, 부상당하고, 나이 들고, 은퇴한다. 클럽은 돈을 벌고 쓰고 파산 위기를 맞는다. 스폰서는 부도난다. 유소년이 데뷔한다. 매 시즌 세계가 달라진다.

2. **설명 가능한 시뮬레이션 (Explainable Simulation)**
   "3-1로 졌다"가 아니라 "23분 실책으로 실점, 51분 주전 CB 부상 이탈, 67분 퇴장" 을 볼 수 있다. 모든 통계는 이벤트 로그에서 파생되므로 숫자와 서사가 어긋나지 않는다.

3. **결정론적 재현성 (Deterministic Reproducibility)**
   동일 시드 + 동일 입력 → 동일 결과. 분쟁 발생 시 경기를 그대로 재현해 검증할 수 있다. 이는 배팅 서비스의 신뢰 기반이다.

4. **깊이 있는 배팅 마켓 (Deep Betting Markets)**
   승/무/패를 넘어 정확한 스코어, 득점자, 카드/코너 프롭, 시즌 우승·승격·강등·득점왕, 플레이오프·컵 우승까지 전 범위.

5. **코드 없는 밸런싱 (Config-driven Balancing)**
   성장률, 부상 확률, 홈 어드밴티지, 포인트 배분, 라운드 간격까지 전부 DB에서 조정. 배포 없이 세계를 튜닝한다.

## 1.6 성공 기준 (KPI)

| ID | 지표 | 목표 | 측정 방법 | 측정 주기 | 관련 요구사항 |
|---|---|---|---|---|---|
| **KPI-1** | 시뮬레이션 무중단 가동률 | 시즌 1주기(≈73시간) 동안 라운드 누락 **0건**, 크론 실행 성공률 **≥ 99.5%** | 크론 실행 로그 / 라운드 완주 카운트 | 시즌마다 | FR-AD-017~022, NFR-CR |
| **KPI-2** | 경기 1건 시뮬레이션 소요 | p95 ≤ **50ms**, p99 ≤ 120ms (서버, 90틱, 라인업 선정 포함) | Vitest 벤치 + 프로덕션 계측 | 상시 | NFR-PF-001 |
| **KPI-3** | 재현성 | 동일 시드 + 동일 입력 재실행 시 이벤트 로그 **100% 일치** | Vitest 시드 스냅샷 테스트 | 커밋마다 | NFR-DT-002, NFR-QA-003 |
| **KPI-4** | 배당 정확도 | 1X2 기준 Brier Score ≤ **0.21** (1,000경기 누적) | 정산 로그 분석 | 시즌마다 | FR-BT-005 |
| **KPI-5** | 라이브 화면 갱신 지연 | 이벤트 발생 → 클라이언트 반영 p95 ≤ **3초** | 클라이언트 계측 | 상시 | FR-UI-022 |
| **KPI-6** | UI 상태 커버리지 | 정의된 전 컴포넌트가 `/sample`에 **4가지 상태**(정상/로딩/빈/에러)로 노출 — 커버율 **100%** | 쇼케이스 체크리스트 | 스프린트마다 | FR-UI-000, FR-UI-001 |
| **KPI-7** | 배팅 정산 정확도 | 오정산 **0건**, 정산 지연 p95 ≤ **30초** | 정산 감사 로그 | 상시 (2차) | FR-BT-011 |
| **KPI-8** | 밸런스 건전성 | ① 시즌당 스폰서 부도율 ≤ **15%** ② 승격팀 5시즌 내 재강등률 **40~70%** ③ 경기당 평균 득점 **2.5~3.0** ④ 홈 승률 **42~48%** | 시즌 밸런스 리포트 + Vitest 분포 테스트 | 시즌마다 | NFR-OB-003, NFR-QA-004 |
| **KPI-9** | 회계 정합성 | 포인트 총량 항등식 위반 **0건** (수입 − 지출 = 잔액 변화) | Vitest 회계 테스트 + 시즌 정산 검증 | 시즌마다 | NFR-QA-005, FR-EC-001 |
| **KPI-10** | 설정 기반 튜닝 | 밸런스 조정 시 **코드 변경 0줄**로 반영 가능한 상수 비율 **≥ 90%** | 공통코드 커버리지 감사 | 분기 | NFR-CFG-001 |
| **KPI-11** | 타입 안전성 | `npx tsc --noEmit` 오류 **0건**, `npm run lint` 경고 **0건** | CI | 커밋마다 | NFR-MT-005 |
| **KPI-12** | 테스트 커버리지 | 엔진 코어(`src/lib/sim/`) 라인 커버리지 **≥ 80%**, 브랜치 **≥ 70%** | Vitest coverage | 커밋마다 | NFR-QA-002 |

## 1.7 범위 밖 (Out of Scope)

| 항목 | 사유 | 재검토 시점 |
|---|---|---|
| 사용자가 팀을 직접 운영/조작하는 매니저 모드 | 제품 정체성(관전+배팅)과 충돌 | 없음 |
| 실화폐 결제 및 환전 | 규제·컴플라이언스 부담. 통화는 포인트 단일 | 3차 |
| 인플레이(경기 중) 배팅 | 결과 선계산 구조와 정보 노출 통제가 복잡 | 3차 |
| 3D/2D 경기 시각화 | 텍스트 중계로 충분. 개발 비용 대비 가치 낮음 | 없음 |
| 실존 선수·클럽·리그 데이터 | 저작권 리스크 (DC-12). 전량 절차적 생성 | 없음 |
| 다중 월드(서버) 병렬 운영 | 단일 월드로 시작 | Q-08 |
| 소셜 기능(댓글·팔로우·랭킹 대결) | MVP 범위 통제 | 3차 |
| 모바일 네이티브 앱 | 반응형 웹으로 대응 | 3차 |
