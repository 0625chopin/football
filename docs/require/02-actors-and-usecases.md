## 2.1 액터 목록

| 액터 ID | 액터명 | 유형 | 설명 | 권한 요약 | 도입 시점 |
|---|---|---|---|---|---|
| **A-1** | 게스트 (Guest) | 인간 | 비로그인 방문자 | 공개 데이터 **읽기 전용**. 배팅 불가 | 1차 |
| **A-2** | 배터 (Bettor) | 인간 | 로그인 사용자 | 읽기 + 배팅 생성 + 자기 지갑/이력 조회 | 2차 |
| **A-3** | 운영자 (Operator / Admin) | 인간 | 내부 관리자 | 시뮬 제어, 공통코드 편집, 시드 관리, 재현 검증, 마켓 강제 정산/보이드, 로그 열람 | 1차(비공개 경로) → 2차(정식 인증) |
| **A-4** | **시뮬레이션 엔진** (Simulation Engine) | **시스템** | Edge Function 크론이 구동하는 자율 프로세스 | 세계 상태 **쓰기 전권** (경기·스탯·성장·이적·재정·수상) | 1차 |
| **A-5** | **배당 산출기** (Odds Engine) | **시스템** | 몬테카를로 프리시뮬 기반 확률·배당 계산기 | 마켓/셀렉션/배당 생성·갱신. 세계 상태는 **읽기만** | 1차(산출) / 2차(마켓 개설) |
| **A-6** | **정산기** (Settlement Engine) | **시스템** | 결과 확정 시 트리거되는 정산 프로세스 | 베팅 상태 및 지갑 원장 쓰기. 세계 상태 읽기만 | 2차 |
| **A-7** | 개발자 (Developer) | 인간 | 구현 담당 | `/sample` 접근, Mock/실데이터 어댑터 토글, 검증 스크립트 실행 | 1차 |

### 액터 권한 경계 (NFR-SEC-003 근거)

```
                    세계 상태(리그/경기/선수/재정)   마켓·배당   베팅·지갑   공통코드
A-1 게스트                 R (시간 필터)              R           -           -
A-2 배터                   R (시간 필터)              R          RW(자기것)    -
A-3 운영자                  R                        RW          RW         RW
A-4 시뮬레이션 엔진          RW (전권)                  -           -          R
A-5 배당 산출기              R                        RW           -          R
A-6 정산기                  R                        R           RW          R
```

> A-4/A-5/A-6은 **논리적 분리**이며 초기 구현에서는 단일 서버 프로세스 내 모듈로 존재해도 무방하다.
> 다만 **DB 권한 경계는 반드시 분리**한다 (NFR-SEC-003).
