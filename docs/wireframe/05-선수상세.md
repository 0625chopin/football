## 2. 화면 목표

> "이 선수가 지금 어떤 상태이고(3초), 지난 5시즌 동안 어떻게 성장하며 어디를 거쳐 왔는가(30초)"를 한 화면에서 답한다.
> **잠재능력(PA) 원값은 어떤 경로로도 알 수 없어야 한다.** 사용자에게 허용된 미래 정보는 **스카우트 등급 ★1~5(범위 표기)** 뿐이다.

### 섹션 구성 — 9개 (일정표 018 기준)

FR-UI-005는 항목을 11개로 열거하지만(프로필 / 레이더 / 컨디션·피로 / 포지션맵 / 몸값·계약 / 시즌별 스탯 / 통산 / 성장곡선 / 부상 타임라인 / 트로피 / 이적이력), 화면 섹션은 **9개**로 묶는다.
- `시즌별 스탯` + `통산` → **E6 스탯 (탭)**
- `트로피` + `이적 이력` → **E9 커리어 이력 (탭)**

## 5. 4상태 명세

FR-UI-005는 **"섹션별 스켈레톤 / 섹션별 빈 문구 / 섹션 단위 격리 + 재시도"** 를 명문화한다. 즉 이 화면은 **페이지 단위 4상태가 아니라 섹션 단위 4상태**다(53일차 완료 판정).

| 상태 | 화면 |
|---|---|
| **Loading** | E1은 라우트 파라미터·헤더 데이터로 **먼저** 렌더(LCP 확보). E2~E9는 **섹션별 `SkeletonBlock`** — 레이더 1블록(정사각), 게이지 2줄, 스탯 행 5개, 곡선 1블록, 타임라인 3행. **각 스켈레톤 높이 = 실제 콘텐츠 높이 고정 → CLS 0**(NFR-PF-009) |
| **Empty** | **섹션별 문구(FR-UI-005 명문)** — E6: **"출전 기록 없음"** / E9-트로피: **"수상 이력 없음"** / E8: **"부상 이력 없음"**. 그 외 파생 섹션: E7 곡선은 시즌 1개 이하면 "성장 데이터가 부족합니다", E9-이적은 "이적 이력 없음". **빈 섹션도 제목은 유지**하고 본문만 `EmptyState`로 대체(레이아웃 붕괴 방지) |
| **Error** | **섹션 단위 격리** — 한 섹션의 실패가 다른 섹션을 무너뜨리지 않는다. 실패 섹션만 `ErrorState` + `[다시 시도]`(해당 섹션만 재요청). **E1 헤더 실패 시에만 페이지 전체 에러**(선수 존재 자체가 불확실) |
| **Success** | 3-1 / 3-2 블록도. E3 가용성 배지·E9 탭은 데이터에 따라 내용만 바뀐다 |

> 은퇴 선수도 상세 조회가 가능해야 한다(FR-PL-015 ③). 이 경우 E3(컨디션·피로)·E5(계약)는 **"은퇴"** 상태 표기로 대체하고 E6~E9는 정상 노출한다. 표기 규약 → **W-28**
