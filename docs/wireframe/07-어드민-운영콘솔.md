# 07. 운영 콘솔 (`/admin`)

> 공통 규약: [`00-공통규약.md`](./00-공통규약.md) — R-1~R-14를 전제로 읽는다.
> 표기 주의: 영역 ID는 `G1`~`G6`(하이픈 없음), 엔티티 ID는 `E-01`(하이픈 있음)이다.
> ⚠️ **이 화면 전체가 NFR-SEC-007 적용 대상이다.** 1차는 인증이 아니라 **비공개 경로 + 환경 플래그**로만 보호되며, 화면 자체에는 로그인/권한 UI를 그리지 않는다(인증 기반 전환은 6팀 037, 49~53일차). 라우트 진입 가드는 미들웨어/배포 설정 영역이며 본 와이어프레임 범위 밖이다.
> ⚠️ **G5(월드 리셋)는 I-13("세계를 리셋하지 말 것")의 직접 적용 지점이다.** 기능은 존재하되 2단계 확인 + 사유 입력을 강제한다.

## 1. 개요

| 항목 | 내용 |
|---|---|
| 라우트 | `/admin` |
| 유스케이스 | **UC-201** 배속 제어, **UC-202** 정지/재개, **UC-203** 시드 조회, **UC-206** 월드 리셋, **UC-207** 로그/감사 이력 열람(엔진 결정 로그 부분) |
| 요구사항 | FR-UI-019(S), FR-AD-001(배속), FR-AD-002(정지/재개), FR-AD-003(시드 계층), FR-AD-005(월드 리셋), FR-AD-007(구조적 로깅), NFR-SEC-007 |
| 주 페르소나 | **A-3 운영자**(내부 관리자, PS-1~4 대상 아님) |
| 담당 Task | 021 (54~55일차) |
| 파일 소유 | `src/app/admin/page.tsx` (5팀) |
| 접근 제어 | **NFR-SEC-007** — 1차: 비공개 경로 + 환경 플래그(변수명은 6팀·1팀과 조정, 예: `ADMIN_CONSOLE_ENABLED`) 보호. 2차: 6팀 037에서 인증·역할 확인으로 전환 |
| ⚠️ 수락 기준 | ① 배속 변경 동작(54일차) ② **2단계 확인 없이 리셋 불가**(55일차) |

## 2. 화면 목표

> **운영자가 진입 3초 내에 "지금 시뮬레이션이 어느 페이즈에서 어떤 배속으로 돌고 있는지, 멈춰 있는지"를 파악하고, 배속·정지·리셋을 실수 없이 통제한다.**
> 이 화면의 조작은 전체 3개 리그의 진행에 직접 영향을 준다(FR-AD-001·002). 위험도가 낮은 조작(배속 조정, 정지/재개)은 마찰 없이, 위험도가 높은 조작(월드 리셋)은 의도적으로 마찰을 높인다.

### 3-2. 데스크톱 (1024+)

```
┌─ [4팀] 전역 헤더 ───────────────────────────────────────────┐
┌─ G1 시뮬 상태 요약 (전폭) ───────────────────────────────────┐
┌─ 좌 컬럼 (44%) ─────────────┐┌─ 우 컬럼 (56%) ────────────────┐
│ G2 배속 제어                ││ G4 시드 조회                    │
│ G3 정지/재개                ││ G5 월드 리셋 (위험 영역, 강조 테두리)│
│                             ││ G6 로그 뷰어 (표, 페이지네이션)  │
└─────────────────────────────┘└─────────────────────────────────┘
```

- 모바일은 단일 컬럼 스택, 순서 G1→G6.
- **G5는 시각적으로 격리**한다 — 다른 카드와 다른 테두리/경고 톤(색상 확정은 4팀 H-11 이후, 지금은 "위험 영역"이라는 구조만 고정)으로, **실수로 다른 조작과 나란히 클릭되지 않게** 한다.
- G6는 데스크톱에서 자체 `overflow-x:auto` 테이블(R-6), 모바일은 카드 리스트.

### G5 상세 — 2단계 확인 + 사유 입력 (I-13 / FR-AD-005 수용 기준 ②)

```
1단계 — [월드 리셋…] 클릭
┌─ 확인 1/2 ──────────────────────────┐
│ ⚠ 정말로 월드를 리셋하시겠습니까?     │
│                                      │
│ 누적 시즌 히스토리는 이 서비스의     │
│ 유일한 강한 해자입니다(I-13).        │
│ 리셋하면 이 자산이 사라집니다.       │
│                                      │
│ 기존 월드:                          │
│  ◉ 아카이브 보존   ○ 완전 삭제       │  ← FR-AD-005 요구사항
│                                      │
│      [취소]           [계속 →]      │
└──────────────────────────────────────┘

2단계 — [계속] 클릭 시
┌─ 확인 2/2 — 최종 ────────────────────┐
│ 새 world_seed (선택, 비우면 자동생성) │
│ [_____________________________]     │
│                                      │
│ 사유 (필수)                          │
│ [_____________________________]     │
│                                      │
│ 계속하려면 아래에 RESET을 입력하세요 │  ← 오타 방지 타이핑 확인
│ [_____]                             │
│                                      │
│      [취소]      [리셋 확정] (비활성*)│
│ * 사유 미입력 또는 RESET 미일치 시 비활성 │
└──────────────────────────────────────┘
```

| # | 규칙 |
|---|---|
| RS-1 | 1단계와 2단계는 **서로 다른 모달**이다(같은 모달의 체크박스 2개 아님) — 오조작 방지 강도를 명확히 다르게 가져간다 |
| RS-2 | 사유 입력은 **필수**(NFR-CFG-002와 동일 원칙 적용). 미입력 시 `[리셋 확정]` 비활성 |
| RS-3 | 리셋 실행은 **E-47 AuditLog**에 `actor_type=HUMAN`, `action=WORLD_RESET`, `payload={reason, archiveOrDelete, newSeed}`로 기록(FR-AD-005 수용 기준 ③) |
| RS-4 | 리셋 중에는 **G1~G4가 즉시 잠금**(중복 클릭 방지) — 완료 후 자동 새로고침 |
| RS-5 | 경고 문구의 "누적 시즌 히스토리가 유일한 강한 해자" 문장은 **I-13 원문 인용**이며 임의로 요약하지 않는다 |

> **FR-AD-008(마켓 수동 정산/보이드)는 이 화면에 그리지 않는다.** ROADMAP Task 021의 3개 화면 체크리스트에 해당 UI가 없고, 배팅 마켓·정산은 이 팀의 040(68~73일차) 소관 도메인이다. 배치 위치(운영 콘솔에 통합 vs `/admin/bet` 신설)는 **W-41**로 남긴다.

## 5. 4상태 명세

| 상태 | 화면 |
|---|---|
| **Loading** | G1은 라우트 진입 즉시 스켈레톤(높이 고정 → CLS 0). G2~G5는 섹션별 `SkeletonBlock`. G6은 **행 스켈레톤 8개** |
| **Empty** | G6: **"로그 없음"**(FR-UI-019 명문 문구). G4 `match_seed` 조회 결과 없음: "해당 경기를 찾을 수 없습니다"(제안, 명문 지정 없음 → **W-42**) |
| **Error** | **섹션 단위 격리.** 각 섹션 `ErrorState` + `[다시 시도]`. **G5(리셋 자체)가 실패하면 페이지 전체를 잠그고 명확한 실패 사유를 표시** — 부분 실패 상태로 방치하지 않음 |
| **Success** | 3-1 / 3-2 블록도. G3 상태·G5 배지는 데이터에 따라 조건부 |
