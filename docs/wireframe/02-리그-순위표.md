- 모바일 축약 열: `# / 팀 / 경 / 승무패(합산 표기) / 득실 / 승점 / 최근5`

- **`#` + 팀명 열은 좌측 sticky** — 가로 스크롤 중에도 어느 팀 행인지 잃지 않는다.

### 존 표기 규약 — NFR-A11Y-002 핵심 (R-4)

| 존 | 아이콘 | 라벨(ko) | 라벨(en) | 색(참고) | 적용 리그 |
|---|---|---|---|---|---|
| 승격 | **▲** | 승격 | Promotion | 초록 | 리그2(1~3위), 리그3(1~3위) |
| 플레이오프 | **◆** | 플레이오프 | Play-off | 파랑 | 리그1(1~10), 리그2(1~4), 리그3(1~2) |
| 잔류 | **─** | (라벨 없음) | — | 중립 | 전 리그 |
| 강등 | **▼** | 강등 | Relegation | 빨강 | 리그1(22~24), 리그2(18~20) |
| 리빌드 제재 | **⚠** | 리빌드 제재 | Rebuild sanction | 주황 | 리그3 최하위 |

> **승격과 플레이오프 존은 겹친다**(예: 리그2 1~3위는 승격이면서 플레이오프 진출). 이 경우 **아이콘 2개 병기(`▲◆`)** 하고 툴팁/스크린리더 텍스트에 두 라벨을 모두 넣는다. → 확인요청 W-07
> 색상은 4팀 H-11 디자인 토큰 수신 후 확정. **와이어프레임 단계에서 hex를 정하지 않는다.**
> 검증: 40일차에 Protanopia / Deuteranopia / Tritanopia 3종 시뮬레이션으로 존 구분 가능 확인.

## 7. 접근성 · 반응형

| 항목 | 대응 |
|---|---|
| **NFR-A11Y-002** | B3-z 존 표기 = **색 + 아이콘 + 텍스트 라벨** 3중. B2 범례 상시 노출. 색맹 시뮬 3종 통과가 40일차 수락 기준 |
| NFR-A11Y-005 | B3에 `<caption>`(시각적으로는 B1이 대체하므로 sr-only 가능), `<th scope="col">` 열 헤더, `<th scope="row">` 팀명. 축약 헤더(경/승/무/패)는 `<abbr title="...">`로 풀네임 제공 |
| NFR-A11Y-003 | 팀 행 = 단일 링크 포커스. `FormStrip` 5칸은 포커스 가능하되 순서를 행 뒤로 |
| NFR-A11Y-004 | 순위표는 실시간 갱신 대상이 아니므로 `aria-live` 불필요 |
| **NFR-RS-002** | B3를 **자체 `overflow-x: auto` 컨테이너**로 감싼다. `#`+팀명 좌측 sticky |
| NFR-RS-001 | 320px에서 body 가로 스크롤 **0** (스크롤은 B3 내부에서만 발생) |
| 폭 깨짐 | ko/en 헤더 길이 차이(`경기`/`P`, `승점`/`Pts`)로 열 폭이 흔들리지 않도록 **열 폭을 숫자 자릿수 기준으로 고정** |
