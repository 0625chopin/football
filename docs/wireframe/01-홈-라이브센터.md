# 01. 홈 / 라이브 센터

> 공통 규약: [`00-공통규약.md`](./00-공통규약.md) — R-1~R-12를 전제로 읽는다.

## 1. 개요

| 항목 | 내용 |
|---|---|
| 라우트 | `/` |
| 유스케이스 | **UC-001** 라이브 경기 목록 보기, **UC-013** 배당 참고 보기(1차) |
| 요구사항 | FR-UI-002(M), FR-UI-022(폴링), FR-BT-014(배당 표시 전용), NFR-PF-009, NFR-RS-001, NFR-A11Y-004 |

## 2. 화면 목표

> **접속 3초 내에 "지금 어느 경기가 진행 중이고 스코어가 몇 대 몇인지"를 파악한다.** (PS-1 성공 신호 = 015 수락 기준)

진행 중 경기가 하나도 없는 시간대(페이즈 전환·프리시즌)가 실제로 존재하므로, **Empty 상태가 정상 상태의 하나**다. 이때는 "다음 킥오프까지 남은 시간"이 화면의 주인공이 된다.

### 상태 배지 표기 (R-4 색상 단독 금지)

| 상태 | 아이콘 | 라벨(ko) | 라벨(en) | 색 |
|---|---|---|---|---|
| 진행 중 | ● (점멸 없음) | `LIVE 67'` | `LIVE 67'` | 강조색 |
| 예정 | ⏱ | `예정 21:30` | `21:30` | 중립 |
| 종료 | ✓ | `종료` | `FT` | 약화 |
| 하프타임 | ⏸ | `HT` | `HT` | 중립 |
| **연장** | ⏱ | `연장 105'` | `AET 105'` | 강조색 |
| **승부차기 진행 중** | ◎ | `승부차기` | `PSO` | 강조색 |

> 연장·승부차기 배지는 **컵·플레이오프 경기**가 라이브 그리드에 노출되므로 필요하다(3개 리그 통합 컵 + 리그별 플레이오프).
> 경과분은 **`minute` + `stoppage` 2필드**다 — `45+2'`로 표기하고 **`47'`로 접지 않는다**(`04-경기상세-라이브중계.md` **E-3**, 2팀 계약).
> 승부차기 스코어 표기는 **규약 R-13**을 따른다 — `2 - 2 (승부차기 4-3)` 형식, 합산 금지. 경과분 `67'`·`105'`는 **엔진 제공값**이며 UI가 계산하지 않는다(규약 R-14 ①).

> 점멸 애니메이션은 `prefers-reduced-motion` 시 제거(NFR-A11Y-006).

## 5. 4상태 명세

| 상태 | 화면 |
|---|---|
| **Loading** | A2에 **카드 스켈레톤 6개**(FR-UI-002 명시). 카드 실제 높이와 **동일 높이 고정** → CLS 0. A1은 서버에서 즉시 렌더(LCP 요소). A3·A4는 행 스켈레톤. |
| **Empty** | A2 자리에 `EmptyState`: **"현재 진행 중인 경기가 없습니다 — 다음 킥오프 HH:MM"** (FR-UI-002 명문 문구). 이때 A1 카운트다운과 A3가 시각적으로 승격되어 화면 주인공이 된다. |
| **Error** | A2 자리에 `ErrorState`: **"라이브 데이터를 불러오지 못했습니다"** + `[다시 시도]`. A1·A3·A4는 **독립 유지**(영역 단위 격리 — A2 실패가 화면 전체를 죽이지 않는다). |
| **Success** | 3-1 / 3-2 블록도. |
