### D-25-1 ESLint 가드레일 2종 무력화 (6팀 발견 → I-140, 해소)

flat config는 같은 파일에 매칭되는 여러 블록이 **같은 규칙 키**를 설정하면 병합하지 않고 **뒤 블록으로 전체 교체**한다. 22일차에 추가된 Task 044 블록(`src/**/*.tsx`)이 앞에 있던 두 블록을 통째로 덮어썼다.

- **H-06 컴포넌트 Supabase 차단** — 컴포넌트에 `@supabase/supabase-js`를 직접 import해도 lint exit 0
- **NFR-DT-001 sim 도메인 `react`/`react-dom`/`@supabase/*` 차단** — 팀장 지시 전수확인에서 추가 발견

즉 규칙 추가 1회가 가드레일 2개를 동시에 죽인 구조적 사고다. 두 블록을 각자 스코프에서 가장 뒤로 재배치하고 하위 패턴을 병합해 수정, **위반 5종 실주입으로 전건 발동 확인**. `--print-config` 대조로 겹치는 다른 조합이 없음도 확인. 무력화 기간 실제 위반 코드는 0건이라 유출은 없다.

### D-25-2 컬러 토큰 배경 대비·색역 미검증 (팀장 실측 발견, 해소)

4팀 초기 산출물은 색맹 3종 상호 구분(ΔE)만 검증했고 **배경 대비는 축 자체가 빠져 있었다.** 팀장이 OKLCH→sRGB→WCAG 상대휘도로 실측한 결과:

- 라이트 `--warning` **1.34:1** — 비텍스트 3:1 기준도 미달
- 다크 `--relegation` 3.70:1 — 본문 텍스트 불가
- **5개 중 4개 토큰이 sRGB 색역 밖** → 브라우저가 클램프하므로 원 ΔE 검증이 실제 렌더 색 기준이 아니었음

조치: 색역 밖 토큰은 채도를 낮춰 색역 안으로 넣었고, `--warning`은 **배지 채움 전용**으로 용도를 확정한 뒤 `--warning-foreground`(배지 위 8.2~9.6:1)를 신설했다. 비텍스트 4색은 라이트/다크 모두 3:1(WCAG 1.4.11) 이상 확보. 채도를 낮추면서 ΔE 하한은 15→**12**로 조정됐다(실측 최소값 기준, 아이콘·라벨 병기 전제와 함께 수용).

### D-25-3 팀장 조율 착오 — 4팀 중복 산출 (해소)

팀장이 4팀을 두 번 소환해 검증 테스트가 `src/app/globals.color.test.ts`와 `src/lib/a11y/contrast.test.ts` **두 파일로 중복 생성**됐다. **팀 잘못이 아니라 팀장 착오다.** 커버 축이 상호 보완적이라 어느 쪽도 버리지 않고 `src/lib/a11y/contrast.test.ts` 한 벌로 통합, `globals.color.test.ts`는 삭제. 통합 후 4축(CVD ΔE / sRGB 색역 / 페이지 배경 3:1 / 배지 쌍 4.5:1) 전부 보존 확인 — 102 케이스 통과. **교훈: 팀원 재소환 전 기존 인스턴스 생존 여부를 확인할 것.**
