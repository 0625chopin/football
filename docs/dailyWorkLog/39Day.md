## 3. 이슈

**신규 3건 등재**(`docs/ISSUES.md`).

- **I-201 — `notFound()`가 HTTP 200을 반환.** 팀장 검증에서 발견. 화면은 404 콘텐츠를 정상 렌더하나 상태 코드가 200이다. 원인은 **라우트마다 `loading.tsx`가 있어 셸이 먼저 스트리밍되고 그 시점에 200이 커밋**되는 것 — 4상태 60파일 구조의 구조적 귀결이며 **5팀 산출물 결함이 아니다**. 완전 미매치 URL은 `global-not-found.tsx`로 정상 404가 나오므로 `notFound()` 호출 페이지에서만 발생한다. 오늘 5팀 페이지가 이 프로젝트에서 `notFound()`를 쓰는 **첫 페이지**라 이제야 드러났다. 프로덕션 빌드 불가(I-62 계열)로 **프로덕션 동작은 미확인**.
- **I-202 — 로거 소비처 배선 미이행.** 로거는 생겼으나 11일차 `fallback.ts`의 "39일차에 교체" 인계 메모가 미이행이라 여전히 `console.warn` 직접 호출이다. 수락 기준은 **API 차원에서는 충족, 저장소 전체 호출부 기준으로는 미충족**.
- **I-203 — 순위표 스켈레톤 행 수를 리그별 팀 수에 못 맞춤.** `loading.tsx`가 `params`를 못 받는 Next 16 제약. 정확한 스켈레톤이 필요하면 페이지 내부 `<Suspense>` 경계로 내려야 한다.

셋 다 **비차단**이다.
