### 6팀 — Task 037 지갑 트랜잭션·낙관적 잠금

`wallet.lock_version`(DB 전용 CAS 토큰 — 도메인 `Wallet`은 8일차 동결이라 미노출) 컬럼과
`wallet_apply_transaction()` RPC 신설. 1콜 = 1트랜잭션(41일차 `tick` 패턴 계승),
잔액부족/버전충돌/락소진을 SQLSTATE(P0001/P0002/P0003)로 구분하고 엣지 함수가 409/404/503로
매핑한다. `supabase/functions/wallet/index.ts`는 검증 + RPC 호출만 하는 얇은 래퍼.
RPC는 `REVOKE … FROM PUBLIC, anon, authenticated` + `GRANT TO service_role`.

**수락 기준은 원격 실측으로 충족** — 임시 auth 유저로 TOPUP 100000 후 PostgREST RPC에
BET_PLACE −10 **100건 동시** 발사 → 100/100 성공, 최종 balance 99000 = lock_version 101 =
원장 101건(중복 0) 전부 일치. 구현 중 `RETURNS TABLE` 출력 컬럼명이 `wallet` 실제 컬럼명과
충돌하는 `42702 ambiguous column` 버그 1건을 발견해 수정 마이그레이션으로 해소했다.

## 3. 이슈

**신규 3건 등재** (`docs/ISSUES.md`).

- **I-267** 지갑 RPC 멱등성 키 부재 — 6팀 / 비차단. 낙관적 잠금은 **동시성**을 막지
  **중복 요청**을 막지 않는다. 타임아웃 후 재전송이 두 번 다 반영된다. 수락 기준(동시 100건)은
  각 요청이 서로 다른 의도임을 전제하므로 실측은 정당하나 "이중 지출 방지"의 나머지 절반이
  남는다. 지금 깨지는 것은 없다(엣지 함수 미배포 + 인증 2차 릴리스라 호출자 0건).
  **적용된 마이그레이션 2건은 수정하지 말고 후속 마이그레이션으로 덧붙일 것.**
  부기: 백오프 `least(0.001 * random(), 0.005)`의 0.005 상한은 도달 불가능한 죽은 식.
- **I-268** `AWARD_PARAM_DEFAULT` 카탈로그 미등록 — 3팀 / 031b. I-121·I-136·I-265와 **같은
  패턴 4회차**. 셋 중 `LEAGUE_TIER_WEIGHT`만 승자를 뒤집을 수 있고 나머지 둘은 영향이 국소적.
- **I-269** 원격 마이그레이션 사전 보고 없이 적용 — 6팀 / 절차. 부작용 0건이나 원격 DB는
  되돌리기 어려운 공유 자원이다. 소환 지시문에 "원격 apply·배포·INSERT는 팀장 승인 후"를
  상시 문구로 고정하고, 원격 적용분의 **같은 턴 내 로컬 1:1 커밋**을 완료 조건에 넣는다.

**이슈로 올리지 않은 판단 1건** — 5팀이 보고한 E9 트로피 빈 문구("트로피 없음")가
와이어프레임 제안("수상 이력 없음")과 다른 건은 **클럽 상세 F7과 공유하는 컴포넌트**라
오늘 범위에서 건드리지 않은 것이 옳다(단독 변경 시 클럽 상세가 함께 바뀐다).
